“When to use GET and POST?” – explain when and why you use one type of request and not the other

Quick summary on some common HTTP methods: For the client to interact with the server in different ways, there are HTTP request methods; and here are a few. The GET method retrieves data from the server. The HEAD method is very similar to the GET method, retrieves data from the server; but only the header information of the server’s response; not the body information. The POST method submits data to the server. The PUT method updates a resource currently on the server.

GET should be used to retrieve or fetch data from a server. It is best used for simple querying and getting resources. Other times GET should be used are for idempotency, bookmarking and caching and performance.POST should be used to send necessary data to the server which will either modify the state of the server or handle some sensitive information. Another times POST should be used are for large data payloads, non idempotent operations and complex data structures.GET should not be used to do POST’s job of handling sensitive information as it could be easy to find through server logs, browser history or referrer headers. GET is also not suitable for large data payloads as URL lengths often have a limit of 2000 characters. GET should also not modify the server state or alter data on the server. POST should not do GET’s job of bookmarking and caching as POST requests are not cacheable and cannot be bookmarked. POST should not be used for performance considerations as they are relatively slower than GET methods due to their inclusion of a request body; compared to GET’s simple data retrieval tasks.
