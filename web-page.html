“How is a web page downloaded?” – use your browser’s network console to load a webpage, and then analyse the sequence of HTTP requests made when loading the page; make this page an annotated explanation of this trace; use a page with a relatively small number of requests (e.g. 10-20) and make sure it has a mixture of images and text files such as style files that are loaded

Using this web page as an example, here’s an analysis of the sequence of HTTP requests that are made when loading the page. First the browser starts off with a request for this HTML document web-page.html, using the GET method. This html file has all the main structure of the page like the text, images and so forth. It also references other resources that will be used on the html such as CSS styling and JavaScript interaction. Next, the browser requests all the CSS style sheets to be used to make this web page look pretty. For this page, it first requests for the css of the text fonts that I imported in, one for the font of the sidebar and header and one for the font of the code that you’re reading right now. Then comes the rest of the CSS files that I used for this assignment, which are called header.css, general.css, sidebar.css and code-text-body.css. Now our webpage is fully styled and looking nice for the customer. Next comes the request for the image files. For my webpage I only have 1 image per page so it's just one jpeg file. Next is the request for the JavaScript file. I used JavaScript in this assignment so I could make the user experience better, by making the sidebar feel better to use so the customer has positive experience navigating my website. If you couldn’t tell already, I tried to make the user interface that is implemented in Visual Studio code.
